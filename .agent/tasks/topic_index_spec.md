# 主題索引生成規範

## 核心原則

主題索引的目的是**提供主題概覽與關鍵資訊摘要**，而非僅列出檔案名稱。使用者應該能夠透過主題索引快速了解：

1. **這個主題涵蓋哪些核心知識**
2. **每個筆記的關鍵洞察是什麼**
3. **筆記之間的關聯性**

## 主題索引應包含的元素

### 1. 主題描述
- 簡潔說明這個資料夾的主題範疇
- 參考 `taxonomy.md` 中的定義

### 2. 筆記列表（含摘要與洞察）
每個筆記應顯示：
- **標題**（WikiLink 格式）
- **摘要**：從筆記的「## 摘要 (Summary)」段落提取第一段
- **關鍵洞察**：從「## 關鍵洞察 (Key Insights)」提取前 2-3 個洞察的標題與首句

### 3. 分組邏輯
- 如果筆記有明確的主題分組（透過 tags），則按主題分組
- 否則按字母順序排列，但仍顯示摘要與洞察

### 4. 交叉引用
- 連結到上層目錄索引
- 連結到同層其他子目錄索引
- 連結到子目錄索引（如有）

### 5. 統計資訊
- 筆記數量
- 主要標籤（顯示前 10 個）
- 最後更新日期

## 格式範例

```markdown
# Geopolitics_Macro 主題索引

本目錄收錄地緣政治事件、宏觀趨勢與投資影響相關筆記。

## 📋 目錄概覽

### [[川普的金流管理學 - 從委內瑞拉到明尼蘇達的戰略邏輯]]

**摘要**: 本文分析川普團隊如何透過掌控委內瑞拉石油資源，實現多重戰略目標：削弱中國能源供應、增加美國石油產量、以及透過金流管理影響國內政治版圖。

**關鍵洞察**:
- 委內瑞拉石油是地緣政治槓桿而非單純能源議題: 控制委內瑞拉石油可同時打擊中國、俄羅斯，並增強美國在全球能源市場的定價權...
- 金流管理是川普的核心執政工具: 透過控制委內瑞拉石油收入流向，川普可以影響美國國內州政府的財政狀況，進而影響政治版圖...

### [[第二供應源投資策略 - 在壟斷陰影下尋找被扶植的備胎]]

**摘要**: 本文提出一個反直覺的投資邏輯：當市場出現絕對壟斷者時，不要只盯著龍頭，而要關注那些被巨頭客戶或政府主動扶植的"第二名"。

**關鍵洞察**:
- 壟斷者的成功會創造第二名的投資機會: 當NVIDIA、SpaceX這類公司越成功，客戶和政府的焦慮就越大，他們會主動用資金、合約、技術支持去扶植競爭者...
- 第二供應源的護城河是客戶的恐懼而非技術優勢: 即使技術稍差、成本更高，只要客戶害怕被壟斷，就會持續買單...

## 🔗 相關主題索引

- [[../Investment_主題索引|⬆️ Investment]] (上層目錄)
- [[../Industry_Analysis/Industry_Analysis_主題索引|Industry_Analysis]]
- [[../Investment_Frameworks/Investment_Frameworks_主題索引|Investment_Frameworks]]

## 📊 統計資訊

- **筆記數量**: 3 篇
- **主要標籤**: 地緣政治, 川普, 委內瑞拉, 供應鏈安全, 第二供應源
- **最後更新**: 2026-01-15
```

## 實作細節

### 摘要提取規則
1. 優先從「## 摘要 (Summary)」段落提取
2. 取第一段文字（到雙換行符號為止）
3. 移除 markdown 格式符號（粗體、斜體、連結）
4. 限制長度在 200-300 字元

### 關鍵洞察提取規則
1. 從「## 關鍵洞察 (Key Insights)」段落提取
2. 提取前 2-3 個編號列表項目
3. 格式：`**標題**: 首句內容...`
4. 每個洞察限制在 100-150 字元

### 分組邏輯
- 如果資料夾內筆記 < 5 篇：不分組，直接列出
- 如果資料夾內筆記 >= 5 篇：
  - 嘗試按第一個 tag 分組
  - 如果分組有意義（>1 組且每組 >1 筆記），則使用分組
  - 否則按字母順序列出

## 維護指南

### 自動生成
執行 `python .agent/generate_indices.py` 會自動：
1. 掃描所有 Garden 目錄
2. 提取每個筆記的摘要與關鍵洞察
3. 生成/更新所有主題索引檔案

### 手動調整
如果自動生成的索引不理想，可以手動編輯：
1. 調整分組邏輯
2. 補充主題描述
3. 添加特殊的交叉引用

### 品質檢查
定期檢查主題索引是否：
- [ ] 提供了有意義的摘要（而非只有檔案名稱）
- [ ] 關鍵洞察能讓讀者快速理解筆記價值
- [ ] 分組邏輯清晰合理
- [ ] 交叉引用完整且有效
