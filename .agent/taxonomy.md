## Usage Guidelines

### Categorization Rules

1. **Keyword Matching**: Score each category based on keyword frequency in title and tags
2. **Highest Score Wins**: Assign to category with highest keyword match count
3. **Minimum Threshold**: If no category scores > 0, assign to Journal (catch-all)
4. **Subcategory Routing**: If category has subcategories, perform secondary matching

### Subcategory Assignment

When a note is assigned to a category with subcategories:
1. Check if subcategory directories exist in `10_Garden/[Category]/`
2. If yes, score each subcategory using its specific keywords
3. Route to subcategory with highest score
4. If no subcategory scores > 0, place in category root

### Structural Principles (applied during Sort Garden)

- **MECE**: Sibling folders must not overlap. A note has exactly one correct home.
- **Pyramid**: Max 3 levels. 3-8 children per parent. Balanced sizing (5:1 max ratio).
- **No Phantom Parents**: Folders with ≤2 direct notes that exist only as routing layers should be eliminated.
- **Leaf Sizing**: Target 5-20 notes. >20 = consider subdivision. >30 = strongly recommend. <3 = merge.
- **Top-Level = Mental Model**: 5-8 categories reflecting how the user thinks, not academic taxonomy.

### Adding New Categories

To add a new category:
1. Add entry to this file with description and keywords
2. Update `config.py` CATEGORIES dictionary
3. Create directory in `10_Garden/[NewCategory]/`
4. Generate topic index: `@Agent Update Indexes`

### Adding New Subcategories

To add subcategories to an existing category:
1. Add subcategory definition to this file
2. Update `config.py` CATEGORIES dictionary
3. Create subdirectories in `10_Garden/[Category]/[Subcategory]/`
4. Run migration: `@Agent Deploy [Category] Split`
